# SOLID Principles
Los principios SOLID se enfocan en hacer el código más legible, escalable y mantenible.

## Single Responsibility Principle
Todos los módulos, clases, funciones, componentes, etc… deben tener una sóla razón para cambiar en el tiempo, y por lo tanto, se encarga de una sola tarea.

## Open / Closed Principle
El Software debe estar abierto para extensión, pero cerrado para modificación. Es decir, se pueden añadir nuevas funcionalidades, pero lo ya existente no se debería modificar.

## Liskov Substitution Principle
Una subclase debería poder sustituir a su clase madre sin alterar el comportamiento esperado.

## Interface Segregation Principle
Es preferible tener muchas interfaces pequeñas que se utilizen por completo, a una interfaz muy grande que se utilice a medias.

## Dependency Inversion Principle
Las clases deberían depender de abstracciones y no de implementaciones específicas.
